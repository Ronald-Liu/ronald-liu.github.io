（重新）迁移到Jekyll

August 09, 2016

Reading time ~1 minute

前传

这博客大概两三年没有更新了，突然感觉是时候重新拾起来了。之前出于对Python的执念强行把生成器从Jekyll换成了Pelican。回头发现真还是Jekyll的社区比较发达，最终又重新换回了Jekyll，真是怨念啊。本篇博文就来说说配置Jekyll写作环境的事情。

配置Jekyll

安装

首先你需要有基本的Ruby运行环境和Gem
之后安装Bundler gem install bundler并且安装Jekyll和它依赖的所有包。

配置网站

Jekyll作为一个静态网站生成器，具有极其功能的同时，配置上也有相当的复杂性。因此建立一个博客最简单的方式就是克隆已有的主题，进行一定的自定义配置，然后将自己的博文直接拷贝进去。
博主使用的主题是HPSTR，可以直接从它的Github主页上克隆，或下载源代码包到本地目录。通过修改_config.yml文件来定制自己的博客
博客文章全部放置在 _posts目录中，可以看看主题自带的一些示例文章来学习具体的语法。
虽然GitHub提供了Jekyll的环境，博主作为控制欲爆棚的汉子还是希望编译的过程运行在本地，也方便前期网站参数的调试。而为了方便将编译完之后的文件推送到GitHub Pages上，可以将自己的Github Pages(路径是xxx.github.io的那个)clone到源文件目录的_site目录中，这样每次进行Jekyll编译之后，只需要在_site目录中执行git add . 和 git push就可以完成Github Pages的更新

配置Emacs

Hyde.el是一个Emacs上用于管理Jekyll博客的Mode。它包含了一个前端，一个管理版本的后端和一个可以对写作过程做提示的Markdown编辑模式。本章介绍在MacOS环境下配置Hyde.el的过程。 请首先确认已经安装好了Markdown Mode

安装

要配置Hyde.el，首先需要下载Hyde.el。解压到emacs.d的目录下。然后在自己的配置文件中添加下面的句子

(add-to-list 'load-path (concat config-directory "/hyde/hyde*.el"))  
(require 'hyde)

博主只维护了一个博客，因此可以在Emacs配置文件中配置一个默认的博客源路径，免得每次启动都要指定这个路径。在配置文件中加入

(setq hyde-home "path/to/your/blog/")  
(setq hyde/jekyll-command "bundle exec jekyll build")

千万注意这个路径最后需要加上’/’，否则会报错 jekyll-command命令并不是必须的，然而在我的MacOS中直接运行jekyll会报依赖错误。由于本人对Ruby实在不了解，于是就用了这个笨办法。

在当前的Hyde.el版本(a8cd6ed)中，存在无法读取默认博客源文件路径的问题，需要修改Hyde.el最后部分

(jekyll-root (or home  
                          (hyde/ask-for-jekyll-root)))

为

(jekyll-root (or hyde-home (or home  
                          (hyde/ask-for-jekyll-root))))

Hyde提供了部署静态网站的快捷键，然而由于不同的网站部署的方式不尽相同，因此需要对Hyde进行对应的配置来让你的网站可以根据需要来进行部署。
前文介绍的Jekyll部署方式使用的是本地编译静态网站，而不是利用Github Pages内置的Jekyll。因此部署工作也就是直接将编译完的网站推送到本人的Github Repo中。为了完成这一点，需要对网站的部署命令进行自定义。方法是在blog源文件主目录内创建.hyde.el文件，并填入下列命令

(setq  hyde/deploy-command   
    "cd path/to/your/blog/_site; git add -A . ; git commit -m autopost`date +%m%d%Y%H%M%S`;git push")

替换其中的path/to/your/blog为你的源文件目录

使用

在需要写博客时，打开Emacs，执行M-x hyde，即可打开Hyde Mode。在设置了默认路径的情况下，不需要输入即可自动进入Major Mode界面。使用光标操作可以选择文件，回车打开文件。

在Major Mode下的主要操作有：

  • n 新建新的草稿
  • c 提交当前文章，输入文章标题时尽量用英文，以便生成英文的文章文件名，进入编辑模式后再修改title字段来修改文章标题
  • P 推送所有未提交的commit（只是版本控制的提交，并非部署)
  • j 运行Jekyll
  • d 部署该博客
  • g 刷新博客
  • p 将一个草稿提交为一个文章
  • q 退出Hyde
  • RET 打开指定文件进行编辑。

在文章编辑Mode下的操作有：

  • C-c C-c 保存并提交文件
  • C-c C-v 预览文件，只能包含Markdown预览，无法对Liquid Tags做预览。
  • C-c C-i 向文章中插入文件，该图片文件会自动拷贝到最终发布的网站中，十分方便
